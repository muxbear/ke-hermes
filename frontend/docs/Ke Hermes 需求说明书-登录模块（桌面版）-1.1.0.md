## 1. 概述

### 1.1 项目背景

本文档定义 Ke-Hermes 一个通用智能体系统的登录与注册页面需求。

### 1.2 适用范围

- PC Web 端：桌面浏览器自适应布局（最小支持 768px 高度全屏展示，无滚动条）

### 1.3 术语定义


| 术语      | 说明                  |
| ------- | ------------------- |
| SSO     | Single Sign-On，单点登录 |
| OAuth   | 开放授权协议，用于第三方登录      |
| SMS     | 短信验证码               |
| CAPTCHA | 图形验证码/人机验证          |
| H5      | 移动端 Web 页面          |

## 2. PC 端登录需求

### 2.1 页面布局与自适应

#### 2.1.1 整体布局

登录页面采用左右分栏布局，全屏高度（100vh），无页面级滚动条。左栏为品牌展示区，右栏为登录操作区：

- **左侧品牌区（40% 宽度）**：
  - 深色科技风渐变背景（`linear-gradient(125.75deg, #050a1f 21.4%, #0a173d 57.1%, #050d29 92.9%)`）
  - 内边距：40px（上下）/ 30px（左右）
  - SVG 几何图形 Logo（96×96px，圆角 20px，蓝紫渐变底色 + 白色多层环形图形 + 发光阴影）
  - 系统名称（"Ke-Hermes"，36px/700，`#ffffff`）
  - 产品定位短文案（"自我进化，越用越强"，20px，标签文字色 `#c7d1e0`）
  - 八项关键特性网格（2 列 × 4 行，水平布局：图标左侧 + 标题/描述右侧）：
    - 自我进化与技能沉淀 — 技能沉淀，越用越强（图标：sparkles）
    - 持久记忆系统 — 三层架构，跨会话延续（图标：database）
    - 丰富的工具生态 — 40+ 工具，MCP 扩展（图标：terminal）
    - 全平台消息网关 — 15+ 平台，指令直达（图标：globe）
    - 安全与隐私 — 沙盒隔离，零数据收集（图标：shield-check）
    - 子代理与并行处理 — 子代理，多线推进（图标：cpu）
    - 调度与自动化 — Cron 定时，自动推送（图标：timer）
    - 模型灵活可切换 — 多提供商，热切换（图标：settings）
  - 特性网格规格：图标 40×40px，标题 18px/700，描述 15px，行间距 24px，列间距 32px，图标与文字间距 14px
  - 背景装饰元素：浮动粒子、呼吸光晕、装饰性科技线条
  - 版权信息（"2026 Ke-Hermes 版权所有"，底部居中，12px）
  - 品牌区垂直布局：双 spacer 居中策略（topSpacer + contentWrapper + bottomSpacer + copyright），各区块间距 40px
- **右侧登录区（60% 宽度）**：
  - 背景色 `#090e1f`，内容垂直居中
  - 登录卡片：最大宽度 520px，背景 `rgba(13,19,41,0.9)`，16px 圆角，1px 半透明边框 + 投影阴影 + 背景模糊（blur 8px）
  - 登录卡片内边距：40px，各元素间距：24px
  - 登录方式切换标签（账号登录 / 手机登录）
  - 对应登录表单（无字段标签，使用输入框内 placeholder 提示）
  - 用户协议勾选
  - 登录按钮
  - 第三方登录入口（5 个彩色图标按钮 + 扫码弹窗）
  - 注册入口链接

#### 2.1.2 响应式断点


| 断点             | 布局变化                                 |
| -------------- | ------------------------------------ |
| >= 1024px      | 左右分栏 40%/60%，完整展示品牌区                 |
| 768px ~ 1023px | 左侧品牌区收窄至 32%；登录区 68%                 |
| < 768px        | 隐藏左侧品牌区，顶部固定栏显示简化 Logo + 标题，登录表单卡片居中 |


#### 2.1.3 视觉设计要点

- **暗色主题**：页面背景 `#060b1a`，卡片背景 `rgba(13,19,41,0.9)`，输入框背景 `#0f172e`
- **品牌色**：`#3b82f6`（主蓝），渐变 `linear-gradient(90deg, #3b82f6, #2563eb)`
- **文字色阶**：主文字 `#f2f5fa`、次要文字 `#8794ad`、标签文字 `#c7d1e0`、弱化文字 `#596680`
- **登录卡片**：最大宽度 520px，圆角 16px，内边距 40px，元素间距 24px
- **输入框**：高度 52px，圆角 10px，边框 `#1f293d`，聚焦时边框变为主蓝色，前缀图标 22px，placeholder 字号 16px
- **主按钮**：高度 52px，蓝色渐变背景 + 发光阴影，圆角 10px，字号 19px/700，禁用态降低不透明度至 40%
- **表单无 label 标签**：输入框内通过 placeholder 文字提示（如"请输入账号"、"请输入密码"，字号 16px）
- **协议复选框**：20×20px，圆角 6px，未选中背景 `#141c33` 边框 `#334059`（1.5px），选中背景 `#3b82f5`
- **"忘记密码"链接**：普通下划线文字链接（`text-decoration: underline`），hover 变为主蓝色，非按钮样式，字号 15px
- **第三方登录图标**：52×52px 圆形（cornerRadius 26px），纯色品牌底色 + 白色文字 + 品牌色投影阴影，图标字号 17px/700
- **品牌区 Logo**：96×96px，圆角 20px，蓝紫渐变底色 + 白色环形图形 + 发光阴影
- **品牌区系统名称**：36px/700，白色
- **品牌区定位文案**：20px，标签文字色
- **品牌区特性网格**：2 列 × 4 行水平布局（图标左 + 标题/描述右），图标 40px，标题 18px/700，描述 15px，行间距 24px，列间距 32px，图标与文字间距 14px
- **品牌区区块间距**：40px（Logo → 名称组 → 特性网格之间）
- **品牌区内边距**：40px（上下）/30px（左右）
- **记住我 + 忘记密码行**：字号 15px，复选框 20×20px
- **协议文字**：字号 14px，协议链接加粗 600，主蓝色
- **第三方登录标签**：字号 15px
- **注册链接行**：字号 15px，"立即注册"加粗 600，主蓝色
- **登录方式标签**：字号 18px，选中态 600 加粗 + 主蓝色 + 底部指示线 36×3px，未选中态普通 + 弱化文字色
- **垂直间距自适应**：关键间距使用 CSS `clamp()` 函数，基于视口高度（vh）动态缩放，确保 100vh 内无滚动条
- **右侧表单区**：`overflow-y: auto` 作为极小视口的兜底滚动
- **字体**：Inter + Noto Sans SC + 系统后备字体

### 2.2 账号密码登录

#### 2.2.1 表单字段


| 字段  | 类型    | 必填  | placeholder | 校验规则                    |
| --- | ----- | --- | ----------- | ----------------------- |
| 账号  | 文本输入框 | 是   | 请输入账号       | 支持用户名/邮箱/手机号；长度 2-64 字符 |
| 密码  | 密码输入框 | 是   | 请输入密码       | 长度 6-12 字符；至少包含字母+数字    |
| 记住我 | 复选框   | 否   | -           | 勾选后 7 天内免登录             |


#### 2.2.2 交互流程

1. 用户输入账号与密码
2. 勾选"同意用户协议"
3. 点击「登录」按钮
4. 前端校验格式 → 后端验证账号密码 → 返回 JWT Token
5. 成功后跳转至系统首页
6. 失败时在输入框下方显示错误信息（如"账号或密码错误"）

#### 2.2.3 安全策略

- 密码传输使用 HTTPS + RSA 前端加密
- 连续 5 次登录失败，锁定账号 15 分钟
- 登录成功后，清除当前账号的失败计数
- 支持验证码兜底：登录失败 3 次后弹出图形验证码（滑动拼图）

### 2.3 手机短信验证码登录

#### 2.3.1 表单字段


| 字段    | 类型           | 必填  | placeholder | 校验规则              |
| ----- | ------------ | --- | ----------- | ----------------- |
| 手机号   | 文本输入框        | 是   | 请输入手机号      | 中国大陆手机号 11 位；1 开头 |
| 短信验证码 | 文本输入框 + 发送按钮 | 是   | 请输入验证码      | 6 位纯数字，有效期 5 分钟   |
| 图形验证码 | 滑动拼图弹窗       | 是   | -           | 获取短信前必须完成滑动拼图验证   |


#### 2.3.2 短信发送流程（关键交互）

```
用户输入手机号
      ↓
点击「获取验证码」
      ↓
弹出滑动拼图验证码弹窗
      ↓
用户拖动滑块完成拼图 → 前端采集轨迹
      ↓
后端校验滑动轨迹有效性
      ↓
校验通过 → 发送短信验证码至手机
校验失败 → 自动刷新拼图，提示用户重试
```

#### 2.3.3 短信发送限制

- 同一手机号 60 秒内只能发送 1 次
- 同一 IP 每小时最多发送 10 次
- 同一手机号每日最多发送 5 次
- 发送间隔内按钮显示倒计时（60s），倒计时结束后可重新发送
- 图形验证码每次发送后自动刷新

#### 2.3.4 登录流程

1. 用户输入手机号
2. 点击「获取验证码」
3. 完成滑动拼图人机校验
4. 接收短信验证码并输入
5. 勾选"同意用户协议"
6. 点击「登录」按钮
7. 后端校验手机号与验证码 → 手机号未注册则自动引导至注册流程
8. 验证通过返回 Token，跳转首页

### 2.4 图形验证码

#### 2.4.1 技术要求

- 类型：滑动拼图验证码（优先滑动拼图，用户体验更好）
- 展示形式：弹窗（Modal），居中显示，带半透明遮罩（`rgba(0,0,0,0.6)`）
- 弹窗背景：`rgba(15,23,46,0.98)`，圆角 16px，1px 半透明边框
- 弹窗标题：「安全验证」
- 刷新机制：验证失败后自动刷新，也可手动点击刷新
- 有效期：生成后 5 分钟内有效
- 候选方案：如滑动验证码不可用，降级为 4 位数字字母混合图片验证码

#### 2.4.2 交互要求

- 登录页（LoginPage）作为验证码弹窗的宿主，统一管理验证码流程
- 验证成功后弹窗自动关闭，触发后续操作（短信发送或登录提交）
- 验证失败后显示错误提示，自动刷新验证码，允许无限重试
- 待处理操作（PendingAction）模式：先记录用户意图（发短信/登录），弹出验证码，验证通过后执行

### 2.5 第三方 OAuth / SSO 登录

#### 2.5.1 支持的第三方登录


| 平台   | 方式       | 图标颜色                     | 适用场景 |
| ---- | -------- | ------------------------ | ---- |
| 微信   | OAuth 扫码 | `#07c160`（绿色）            | 个人用户 |
| 支付宝  | OAuth 扫码 | `#1777f9`（蓝色）            | 个人用户 |
| 飞书   | SSO      | `#3370fa`（蓝紫色）           | 企业用户 |
| 钉钉   | SSO      | `#0089f9`（天蓝）            | 企业用户 |
| 企业微信 | SSO      | `rgba(7,193,96,0.85)`（绿） | 企业用户 |


#### 2.5.2 交互设计

- 在登录卡片内，分割线下方展示第三方登录入口区域
- 布局：5 个纯色圆形图标按钮（52×52px，cornerRadius 26px），白色文字，水平均匀排列，间距 20px
- 每个图标带品牌色投影阴影（如 `box-shadow: 0px 3px 8px rgba(7,193,96,0.25)`）
- hover 时图标上浮 2px + 阴影加深
- 所有第三方登录点击后均弹出二维码扫码弹窗（Modal），用户扫码授权
- 所有第三方登录均需绑定系统账号：
  - 首次登录：引导绑定已有账号或创建新账号
  - 已绑定：直接登录

#### 2.5.3 OAuth 回调安全

- 回调 URL 必须使用 HTTPS
- 校验 state 参数防止 CSRF 攻击
- 回调后验证 authorization code 有效性
- 首次第三方登录需额外校验手机号或邮箱以完成账号绑定

### 2.6 用户协议

#### 2.6.1 展示要求

- 登录按钮上方展示协议勾选区
- 布局：复选框（20×20px，圆角 6px） + 「我已阅读并同意《用户服务协议》和《隐私政策》」
- 协议名称使用可点击链接样式（主蓝色，加粗 600），点击后在弹窗中展示协议全文
- 复选框默认为未勾选状态

#### 2.6.2 交互逻辑

- 未勾选协议时点击登录，复选框区域红色高亮闪烁提示
- 未勾选协议时登录按钮仍可点击（由业务逻辑拦截并高亮提示）
- 协议弹窗内提供「同意」和「不同意」按钮：
  - 点击「同意」：自动勾选外部复选框，记录协议版本号，关闭弹窗
  - 点击「不同意」：保持未勾选状态，关闭弹窗

#### 2.6.3 协议版本管理

- 后端存储用户同意协议的版本号与时间戳
- 协议更新后，已登录用户下次访问时需重新同意新版本
- 历史协议版本需保留并可追溯

## 3. PC 端注册需求

### 3.1 注册方式

支持三种注册途径：


| 途径        | 入口            | 说明     |
| --------- | ------------- | ------ |
| 手机号注册     | 登录页底部「立即注册」链接 | 主注册流程  |
| 邮箱注册      | 登录页底部「邮箱注册」   | 备选注册流程 |
| 第三方账号关联注册 | OAuth 首次登录引导  | 快速注册   |


### 3.2 手机号注册流程

#### 3.2.1 表单字段


| 字段    | 类型        | 必填  | 校验规则                       |
| ----- | --------- | --- | -------------------------- |
| 手机号   | 文本输入框     | 是   | 11 位中国大陆手机号                |
| 短信验证码 | 文本 + 发送按钮 | 是   | 6 位数字，有效期 5 分钟             |
| 图形验证码 | 滑动拼图弹窗    | 是   | 获取短信前必须完成滑动拼图验证            |
| 昵称    | 文本输入框     | 是   | 2-20 字符；支持中英文数字下划线         |
| 密码    | 密码输入框     | 是   | 8-32 字符，含大小写字母+数字+特殊字符至少三种 |
| 确认密码  | 密码输入框     | 是   | 与密码一致                      |
| 用户协议  | 复选框       | 是   | 必须勾选                       |


#### 3.2.2 注册流程

```
注册页入口（登录页底部链接）
        ↓
输入手机号
        ↓
图形验证码校验（滑动拼图弹窗）
        ↓
发送短信验证码（60s 倒计时）
        ↓
输入短信验证码、昵称、密码、确认密码
        ↓
勾选用户协议
        ↓
点击「注册」
        ↓
前端格式校验 → 后端验证码校验 → 创建账号
        ↓
注册成功 → 自动登录 → 跳转首页
```

#### 3.2.3 状态处理

- 手机号已注册：提示「该手机号已注册，是否直接登录？」并提供跳转登录链接
- 验证码错误：提示重新输入，每日最多错误 5 次，超过后该手机号当日禁止注册
- 网络异常：提示「网络异常，请稍后重试」

### 3.3 邮箱注册流程

与手机号注册类似，差异如下：

- 将手机号替换为邮箱地址
- 将短信验证码替换为邮箱验证码（发送至邮箱的 6 位数字验证码）
- 邮箱验证码有效期 10 分钟
- 图形验证码同样为必填项

### 3.4 注册后行为

- 自动创建用户默认工作空间
- 生成系统默认头像（昵称首字）
- 弹出新手引导（可选跳过）
- 发送欢迎通知（站内消息）

## 4. 通用功能需求

### 4.1 路由与导航


| 路由                | 页面        | 布局组件       |
| ----------------- | --------- | ---------- |
| `/login`          | PC 登录页    | AuthLayout |
| `/register`       | PC 注册页    | AuthLayout |
| `/register/email` | PC 邮箱注册页  | AuthLayout |
| `/oauth/callback` | OAuth 回调页 | 独立页面       |
| `/`               | 首页（需登录）   | 独立页面       |


- 未登录用户访问任何需认证页面时，自动重定向至 `/login`，登录成功后回跳原页面
- 登录/注册路由嵌套在 AuthLayout 下，共享品牌区 + 表单区布局

### 4.2 验证码通用规则


| 类型    | 长度    | 有效期   | 重发间隔 | 每日上限    |
| ----- | ----- | ----- | ---- | ------- |
| 短信验证码 | 6 位数字 | 5 分钟  | 60 秒 | 5 次/手机号 |
| 郵箱验证码 | 6 位数字 | 10 分钟 | 60 秒 | 10 次/邮箱 |
| 图形验证码 | 滑动拼图  | 5 分钟  | 无限制  | 无限制     |


### 4.3 密码策略

- 长度 8-32 字符
- 必须包含以下四类中的至少三类：大写字母、小写字母、数字、特殊字符
- 不允许与用户名/邮箱/手机号相同
- 不允许使用常见弱密码（如 12345678、password 等，维护黑名单）
- 密码传输使用 RSA 非对称加密

### 4.4 Session 与 Token 管理

- JWT Token 模式，Access Token 有效期 2 小时，Refresh Token 有效期 7 天
- Access Token 过期后自动使用 Refresh Token 续期（去重锁机制防止并发刷新）
- Refresh Token 过期后跳转登录页
- 单设备登录或多设备登录策略由后端配置决定（默认允许多设备）
- 用户主动退出时，Token 即时失效

### 4.5 设备与终端识别

- PC 端独立账号体系
- 后端需识别请求来源终端类型（User-Agent 解析）
- 登录历史需展示设备类型、IP 地址、登录时间

### 5.3 安全性

- 全站 HTTPS，HSTS 头部
- Content-Security-Policy 头部设置

- 敏感信息（密码、验证码）不在 URL 参数中传递
- 前端不存储明文密码，仅存储 Token
- 日志中脱敏处理手机号、邮箱等个人信息

- 登录按钮支持 Enter 键触发

- 页面加载性能监控（Core Web Vitals）
