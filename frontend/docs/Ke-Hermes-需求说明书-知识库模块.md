# ke-hermes 需求说明书 - 知识库模块

## 1. 模块概述

知识库模块是 ke-hermes 智能体服务平台的核心组件之一，面向企业级多模态 RAG（Retrieval-Augmented Generation）系统设计。该模块负责管理知识库的全生命周期，包括知识库创建与配置、多模态文档上传与管理、文档索引流水线监控、知识图谱可视化、混合检索调试等功能。

### 1.1 RAG 系统背景

本知识库模块基于以下 RAG 技术架构：

- **多模态文档解析**：支持 PDF、Word (docx)、Markdown、CSV、HTML、图片（OCR）等格式
- **混合检索**：稠密向量检索 + 稀疏 BM25 倒排索引融合，权重可调
- **知识图谱增强**：基于 LLM 的实体关系抽取，构建结构化知识网络
- **Reranker 精排**：对召回结果进行二次排序，提升检索精度
- **多索引策略**：支持固定长度、递归分隔符、语义分块、Markdown 结构、Agentic 智能等多种切片策略

## 3. 功能需求

### F1: 知识库列表管理

提供知识库列表页面，支持网格/列表两种视图模式。

**具体要求：**

- 页面顶部展示知识库统计卡片：知识库总数、文档总数、分片总数、实体总数、索引中任务数
- 搜索栏支持按名称、描述、标签检索知识库
- 网格视图：3 列卡片布局，每张卡片展示知识库名称、状态、描述、标签、指标（文档/分片/实体/关系）、索引配置概要
- 列表视图：表格布局，列包含名称、状态、文档数、分片数、实体/关系数、大小、索引方案、标签、更新时间、操作
- 网格末尾提供"新建知识库"占位卡片
- 空状态时显示引导提示

### F2: 知识库 CRUD

**具体要求：**

- **创建**：对话框表单，填写名称（必填）、描述、标签（逗号分隔），同时配置初始索引方案（切片算法、Embedding 模型、稀疏检索算法、实体抽取模型、是否启用知识图谱）
- **编辑**：支持修改知识库名称、描述、标签
- **删除**：二次确认对话框，提示将永久删除知识库及其全部文档、向量、图谱数据
- 操作结果通过 Element Plus Message 反馈

### F3: 知识库详情 - 概览 Tab

进入知识库详情后默认展示概览信息。

**具体要求：**

- 统计卡片：文档总数、分片数、实体数、关系数
- 最近索引活动列表：展示最近 5 条文档的索引进度（进度条 + 状态标签）
- 当前索引方案摘要：切片策略、Embedding 模型、稀疏检索、实体/关系抽取模型、Reranker、Top-K/混合权重
- 标签展示

### F4: 知识库详情 - 文档管理 Tab

管理知识库内的文档，支持上传与索引流水线监控。

**具体要求：**

- 文档搜索（按文件名检索）
- 文档列表表格：文档名（含类型图标）、大小、分片数、实体/关系数、状态（带进度条）、操作（重试/编辑/删除）
- **上传文档**对话框：支持拖拽区域 + 快速添加示例文件按钮（PDF/MD/DOCX/CSV/HTML/IMAGE）
- 点击文档行，右侧展开**索引流水线**面板：
  - 总进度条
  - 分片/实体/关系统计
  - 8 阶段流水线：排队 → 解析 → 切片 → 向量化 → BM25 倒排 → 实体抽取 → 关系抽取 → 入库
  - 每阶段显示状态图标（等待/运行中/完成/失败）和进度条

### F5: 知识库详情 - 知识图谱 Tab

以 SVG 可视化方式展示文档中抽取的实体与关系。

**具体要求：**

- SVG 画布渲染实体节点和关系连线
- 实体节点大小与提及次数成正比
- 关系连线带有标签文字
- 实体类型用不同颜色区分（人物/组织/产品/概念/算法）
- hover 实体/关系时高亮关联元素
- 左侧实体列表：按提及次数排序，hover 联动高亮
- 按实体类型筛选
- 图例展示

### F6: 知识库详情 - 检索测试 Tab

在知识库内进行检索测试，验证索引效果。

**具体要求：**

- 三种检索模式切换：混合检索（向量+BM25 融合）、向量检索（稠密语义）、BM25（稀疏关键词）
- 模式卡片展示模式名称和描述，选中时高亮
- 查询输入框，支持回车触发检索
- 显示当前检索配置信息（Embedding 模型、Top-K、Reranker）
- 检索结果列表：排名、来源文档、综合分数、向量分数、BM25 分数、内容片段（查询词高亮）
- 空状态提示

### F7: 知识库详情 - 索引配置 Tab

配置知识库的索引参数，支持实时预览。

**具体要求：**

分以下配置区域，每个区域用卡片包裹：

1. **文档切片**
   - 切片算法下拉选择（固定长度/递归分隔符/语义分块/Markdown 结构/Agentic 智能），每项带描述
   - 分片大小滑块（128-2048 tokens）
   - 重叠大小滑块（0-512 tokens）

2. **向量化 (Embedding)**
   - Embedding 模型选择（BGE-Large-zh v1.5 / BGE-M3 / OpenAI text-embedding-3-large/small / Qwen3-Embedding-8B / Jina Embeddings v3），每项显示维度

3. **稀疏检索 (BM25)**
   - 算法选择（BM25 / BM25+ / TF-IDF / 不启用）
   - BM25 参数调整：k1（词频饱和 0.5-3.0）、b（长度归一化 0-1）

4. **知识图谱抽取**
   - 启用/禁用开关
   - 实体抽取模型选择
   - 关系抽取模型选择

5. **检索与重排**
   - Top-K 滑块（1-50）
   - 混合权重 α 滑块（0-1，向量↔BM25 权重）
   - Reranker 启用/禁用开关
   - Reranker 模型选择

- 右侧粘性配置预览面板：实时显示当前配置摘要
- "保存并重新索引"按钮（含保存成功反馈）
- "恢复"按钮（撤销修改）

### F8: 索引动画

- 文档状态为非终止态（非 queued/failed/indexed）时，进度以一定间隔自动推进
- 索引中状态的知识库卡片和文档状态标签显示动态效果

## 4. 页面结构

```
知识库列表页 (KnowledgeBaseView, selectedKb === null)
├── 页面标题栏（标题 + 描述 + 新建按钮）
├── 统计卡片行（5 个 KbStatCard）
├── 搜索筛选栏（搜索框 + 筛选按钮 + 视图切换）
├── 知识库网格/列表
│   ├── KbCard × N
│   └── 新建占位卡片
├── KbCreateDialog（新建对话框）
└── 删除确认对话框

知识库详情页 (KnowledgeBaseView, selectedKb !== null)
└── KbDetail
    ├── 粘性顶栏（返回按钮 + 名称 + 状态 + 操作按钮）
    └── Tabs
        ├── 概览 (KbOverviewTab)
        ├── 文档 (KbDocsTab)
        ├── 知识图谱 (KbGraphTab)
        ├── 检索 (KbSearchTab)
        └── 索引配置 (KbConfigTab)
```

## 5. 数据类型

见 `src/types/knowledgeBase.ts`，核心类型包括：

| 类型         | 说明                                          |
| ------------ | --------------------------------------------- |
| `KBStatus`   | 知识库状态：ready / indexing / error / draft  |
| `DocStatus`  | 文档索引状态：8 阶段 + queued / failed        |
| `ChunkStrategy` | 切片策略：fixed / recursive / semantic / markdown / agentic |
| `SparseAlgo` | 稀疏检索算法：bm25 / bm25_plus / tf_idf / none |
| `IndexConfig`| 索引配置完整参数                              |
| `KBDoc`      | 知识库文档（含索引阶段和流水线信息）           |
| `Entity` / `Relation` | 知识图谱实体与关系                  |
| `KB`         | 知识库完整信息（含文档列表、图谱数据、配置）    |

## 7. 交互流程

1. 用户点击左侧菜单"知识库" → 进入知识库列表页
2. 列表页搜索/筛选知识库 → 点击"新建知识库"或点击卡片进入详情
3. 新建知识库：填写信息 → 配置索引方案 → 创建 → 自动进入该知识库详情
4. 知识库详情：通过 Tab 切换查看概览/文档/图谱/检索/配置
5. 文档管理：上传文档 → 查看索引流水线 → 等待索引完成
6. 检索测试：选择模式 → 输入查询 → 查看检索结果
7. 索引配置：调整参数 → 保存并重新索引

## 8. 第一阶段实现说明

第一阶段使用前端 Mock 数据实现完整 UI 交互，后续对接后端 API 时仅需替换 `src/services/knowledgeBaseApi.ts` 中的函数实现。Mock 数据包含 5 个示例知识库、6 个示例文档、8 个示例实体、8 条示例关系，覆盖所有状态和索引阶段。
